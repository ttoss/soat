# SOAT Website

Docusaurus site serving https://soat.ttoss.dev.

## Local Development

```bash
pnpm run dev
```

This command runs all doc generators (`generate-all`) and starts a local development server. Most changes are reflected live without having to restart the server.

```bash
pnpm run build
```

Builds the production site. The build fails on broken internal links or anchors (`onBrokenLinks: 'throw'`).

## Website Structure

The navbar has three documentation tabs, each backed by a sidebar defined in `sidebars.ts`. A tab answers "what mode is the reader in?" — learning, doing, or looking something up:

- **Docs** (`docsSidebar`) — learning-oriented content:
  - **Introduction** — what SOAT is and the "one backend, four surfaces" model.
  - **Getting Started** — quick start, key concepts, and choosing a client surface.
  - **Client Surfaces** — setup and usage guides for the SDK, CLI, and MCP server.
  - **Platform** — one page per module (`docs/modules/*.md`), grouped into: Identity & Access, Storage & Retrieval, Agents & Conversations, Orchestration & Automation, Declarative Deployment, and Operations. The grouping mirrors `introduction.md`; `docs/modules/index.md` is the category landing page.
  - **Self-Hosting & Production** — environment variables and production configuration.
- **Tutorials** (`tutorialSidebar`) — end-to-end walkthroughs (`docs/tutorials/*.md`), grouped by theme: Basics, Agents, Knowledge & RAG, Orchestration & Multi-Agent, Automation & Deployment. Tutorial files are also executed by CI (`tests/run-tutorials.sh`), so keep their CLI code blocks runnable.
- **Reference** (`referenceSidebar`) — generated and lookup-oriented material: REST API (OpenAPI-generated), SDK Services, CLI Commands, MCP Tools, Formation Types, Expressions & Templating, Permissions Reference, and OpenAPI Specs.

Module and tutorial sidebars are **explicit** (not autogenerated). When adding a new module or tutorial page, add its doc id to the matching category in `sidebars.ts`.

## Generated docs

`pnpm generate-all` produces the gitignored reference content:

| Script | Output |
| --- | --- |
| `generate-openapi-specs-page` | `docs/openapi-specs.md` |
| `generate-api-docs` (openapi plugin) | `docs/api/**` |
| `generate-mcp-tools-docs` | `docs/mcp/tools.md` |
| `generate-sdk-services-docs` | `docs/sdk/services.md` |
| `generate-cli-commands-docs` | `docs/cli/commands.md`, `docs/cli/commands/*.md` |
| `generate-permissions-page` | `docs/permissions.md` |
| `generate-formations-resource-docs` | `docs/formations-types/*.md` |

Never edit generated files by hand — change the source (OpenAPI specs, permissions JSON, formation modules) and regenerate.

## Authoring rules

Module page structure, cross-referencing rules, and the docs drift guardrails (`scripts/docs-lint.mjs`, ID-prefix checks) are documented in `.claude/rules/website.md`. Follow them for any docs change.
